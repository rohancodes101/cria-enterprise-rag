# CRIA — Change Record Intelligent Assistant
### AI-Assisted Change Request Generation Using Historical Operational Knowledge

CRIA is an AI-powered system designed to simplify the process of creating structured change requests in enterprise environments. By leveraging **Retrieval-Augmented Generation (RAG)**, CRIA transforms natural language descriptions into professional, audit-ready change documentation based on proven historical data.

---

## 📖 Overview

In many organizations, engineers spend significant time drafting change records. While similar changes have often been executed before, finding and reusing that operational knowledge is a manual, time-consuming challenge. 

**CRIA addresses this by:**
* Learning from historical change records.
* Generating structured requests automatically.
* Ensuring consistency and reliability across documentation.

Instead of writing from scratch, an engineer describes the change in plain English, and CRIA retrieves similar historical records to generate:
* **Short Description**
* **Implementation Plan**
* **Risk and Impact Analysis**
* **Test Plan**
* **Backout Plan**
* **Justification**

---

## 🚀 Why This Project Exists

Creating change requests is a critical but repetitive compliance activity. Traditionally, this involves:
1.  Searching through legacy ticket systems (ServiceNow, Jira, etc.).
2.  Consulting senior colleagues for tribal knowledge.
3.  Validating approaches with domain experts.
4.  Multiple iterations of documentation before approval.

CRIA was built to explore how AI can ground its responses in **real operational data** rather than hallucinating generic steps, providing a reliable starting point for any infrastructure or software change.

## 🧠 Key Idea

> "If a change has already been implemented in the past, its operational knowledge should be reusable."

By treating historical records as a live knowledge base, CRIA ensures that step-by-step procedures, risk mitigations, and rollback instructions are based on what actually worked in the past.

## 🏗️ Core Architecture

CRIA utilizes a **Retrieval-Augmented Generation (RAG)** architecture. This avoids the need for constant model retraining and ensures data privacy and traceability.



### System Workflow
1.  **User Input:** Natural language description.
2.  **Semantic Embedding:** Conversion of text to numerical vectors.
3.  **Vector Search:** Finding the most relevant historical records via FAISS.
4.  **Context Construction:** Building a prompt grounded in retrieved data.
5.  **LLM Generation:** Producing the structured text.
6.  **Output Structuring:** Formatting the final record for the user.

---

## 🛠️ Technical Stack & Implementation

### 1. Semantic Embedding Generation
The system converts queries into vectors using a multilingual, high-performance model:
* **Model:** `granite-embedding-278m-multilingual-Q4_K_M.gguf`
* **Function:** Captures the semantic intent behind the engineer's request.

### 2. Vector Similarity Search
Historical records are indexed using **FAISS** (Facebook AI Similarity Search).
* **Efficiency:** High-speed similarity search in high-dimensional spaces.
* **Retrieval:** Matches the current request against the top $k$ most similar past changes.

### 3. Structured Generation
The final output is generated by a localized, instruction-tuned LLM:
* **Model:** `granite-3.3-2b-instruct-Q4_K_M.gguf`
* **Capability:** Synthesizes the retrieved context into a coherent, structured plan.

### 4. Local AI Inference
A core design goal was **security and privacy**. 
* **No Cloud Dependency:** Runs locally on CPU using quantized GGUF formats.
* **Air-Gapped Potential:** No internet connectivity required after the initial setup.
* **Data Sovereignty:** Sensitive operational procedures never leave the internal network.

# The internal logic flow of CRIA
query = user_input()

# Convert input to vector
query_embedding = embedding_model.embed(query)

# Retrieve top 3 similar historical records

context = build_context(similar_records)

# Generate and parse the result
response = generation_model.generate(prompt)

## 💡 Advantages of This Approach

CRIA provides several high-value benefits compared to traditional manual workflows:

* **Faster Drafting:** Significant reduction in the time required to draft structured change requests.
* **Knowledge Reuse:** Systematically reuses historical operational knowledge that would otherwise be "siloed."
* **Consistency:** Standardizes documentation format and technical depth across different teams.
* **Explainability:** Outputs are grounded in real-world historical records, making them traceable and justifiable.
* **Search Optimization:** Reduces time spent manually hunting through old change tickets in legacy systems.
* **Error Reduction:** By providing a strong starting draft, it minimizes the risk of forgetting critical validation or backout steps.
